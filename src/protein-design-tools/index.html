    Community health (CHAOSS)

    CHAOSS metrics reference ↗

    Per-metric comparison of every tool, in the spirit of the CHAOSS project (Community Health Analytics in Open Source Software). Each small chart is one metric; bars are tools, sorted, colored by category. Commit-derived metrics (commits, code authors, bus factor, churn, project age, recency) come from the GrimoireLab git enrichment in OpenSearch; engagement (PRs, issues, reviews) from Neo4j; and reach (stars, forks, contributors, affiliations) from the RDF graph. Hover a bar for the value and a tool card.

    Hover a tool to trace it across every chart · click to pin · double-click to open.

    Relationship explorer

    Each dot is a tool, colored by category. Dashed lines mark the median of each axis (four quadrants). Count axes use a log scale automatically. Hover/click a dot — it links to the bar charts below.

    Metrics over time

    One line per tool, from per-commit timestamps in OpenSearch (GrimoireLab). Hover or pin a tool to trace its trajectory — pins are shared with every chart on this page.

    Bus factor = fewest top committers whose work covers >50% of commits (lower ⇒ more key-person risk). Recency = months since the last commit (lower ⇒ more actively maintained). Tools with no GrimoireLab commit data are omitted from commit-derived charts only.

    Contributor-overlap network

    View

    Edges connect tools that share any author in their op:Contribution records. Edge thickness is the number of shared contributors. Force: drag nodes to pin them, click to open the modal. Hive: each axis is one of the 5 categories ("waves" from the Step-1 brief), nodes ordered by stars along the axis.

    Cross-tool contributors

    Authors who appear in two or more tools. ORCID identities are mapped to GitHub via schema:url. Affiliations come from org:hasMembership (ROR only). Each tool chip is colored by category — click a chip to open that tool's modal.

    Cross-tool organizations

    ROR-identified institutions reached by following each contributor's org:hasMembership → org:organization chain (with the ORCID → GitHub bridge applied). One row per institution, aggregated across all tools where at least one of its affiliates contributed.

    Institution map

    Bipartite graph: institutions (ROR) linked to the tools their affiliated contributors worked on. Edge thickness = number of shared contributors. Drag to pin, click a tool to open it.

    Dependency ecosystem

    View

    Outgoing DEPENDS_ON edges from the seed tools, per Neo4j. Seed nodes are colored by category; dependency-only nodes are gray and sized by how many seeds depend on them. Hover for the seeds sharing each library; click a node to open its modal (when it's also a seed).

    Sources: Oxigraph hybrid graph https://open-pulse.epfl.ch/graph/2026-05/hybrid · Open Pulse Neo4j · README excerpts from raw.githubusercontent.com. CSV exports in data/. Rebuild with python src/protein-dashboard/build.py.
